# 状态机与动作流转

## 1. 核心原则

选品系统不是数据展示器，而是带流转约束的决策工具。

所以每一条候选都必须有状态，不允许长期停留在“看了但没动作”。

---

## 2. 主状态机

| 状态 | 含义 | 进入条件 | 下一步 |
|------|------|----------|--------|
| 待评估 | 刚导入，还没判案 | CSV 新导入 | 观察池 / 淘汰 |
| 观察池 | 值得盯，但暂不投入 | RPS 60-77 或趋势未确认 | 深挖 / 淘汰 |
| 待补证 | 信息不够，不能拍板 | RPS 40-59 或关键字段缺失 | 观察池 / 淘汰 |
| 供应链核价 | 已明确值得推进 | RPS >= 78 且破局点明确 | 打样 / 淘汰 |
| 打样验证 | 进入实际验证 | 核价通过 | 小单测试 / 淘汰 |
| 小单测试 | 接近商业化验证 | 打样通过 | 正式立项 / 淘汰 |
| 淘汰库 | 明确不做 | 一票否决 / 分数过低 | 归档 |

---

## 3. 状态切换门槛

## 待评估 -> 观察池
必须满足：
- 未触发一票否决
- RPS >= 60
- 至少有基础需求证据

## 待评估 -> 待补证
满足任一：
- RPS 40-59
- 关键字段缺失较多
- 结论方向不清晰

## 待评估 -> 淘汰库
满足任一：
- Eligibility Gate reject
- RPS < 40

## 观察池 -> 供应链核价
必须满足：
- RPS >= 78
- 至少 1 个明确破局点
- 高频差评点可工程化或文案化解决

## 待补证 -> 观察池
必须满足：
- 补齐关键字段
- 重新评分 >= 60

## 供应链核价 -> 打样验证
必须满足：
- 核价可接受
- 利润率达标
- 供应链可执行

## 任意状态 -> 淘汰库
必须记录：
- 淘汰原因
- 证据
- 决定人
- 时间

---

## 4. 淘汰原因标签

建议固定以下标签，不要自由发挥：

- 高危禁入
- 高需低利陷阱
- 创新空间锁死
- 供应链无破局点
- 竞争过热
- 垄断过强
- 数据不足
- 老板手动否决

---

## 5. 人工干预规则

允许人工强行推进，但必须留痕：

### 强推字段
- 强行推进原因
- 决策人
- 时间
- 证据链接 / 备注

### 目的
不是禁止拍脑袋，而是让拍脑袋也能复盘。

---

## 6. 前端交互建议

最合适的是 **看板（Kanban）**：

- 列 = 状态
- 卡片 = 产品方向
- 卡片上显示：
  - 产品方向
  - RPS 分数
  - 风险标签
  - 下一步动作
  - 更新时间

这样团队看一眼就知道：
- 哪些在观察
- 哪些该去找工厂
- 哪些必须放弃

---

## 7. 本质

这个状态机不是形式主义，它的价值是：

> **把“分析”强行推进成“动作”。**
