# RPS 规则表 v1（Gemini 重构版）

RPS = **Recon Priority Score（侦察优先级分数）**

重构后的原则很简单：

> **先 Gate，再线性打分；高风险不做乘法，信息不足不偷偷折分。**

---

## 1. 总体结构

```text
Step 0: Eligibility Gate
Step 1: 计算 RPS_base
Step 2: 加人工 override
Step 3: 扣固定风险分
Step 4: 打标签（过热 / 创新空间锁死 / 数据过期）
Step 5: 映射到动作状态
```

### 本次删掉的旧逻辑
- 删除 `riskMultiplier`
- 删除 `confidenceMultiplier` 对总分的直接折损
- 删除 `overheatingPenalty` 作为扣分项

保留的思想：
- 风险仍然重要，但改成 **固定扣分 / 直接拦截**
- 信息缺失仍然重要，但改成 **待补证状态**
- 过热仍然重要，但改成 **标签，不直接扣分**

---

## 2. Step 0：Eligibility Gate（先判能不能做）

### v1 当前可执行的拦截条件

| 条件 | 结果 |
|------|------|
| overallRisk = 极高 | reject |
| 初筛结论命中侵权/危险品/禁入/专利雷区 | reject |
| currentRating > 4.6 且 reviewCount > 2000 且 高频差评点为空 | reject |

### 说明
第三条本质是：
- 市场已经成熟
- 评论壁垒很高
- 看不到明确可改造痛点

这类不适合继续浪费时间。

### v2 待补的 Gate
| 条件 | 结果 |
|------|------|
| contributionMarginRate < 阈值 | reject |
| estimatedUnitMargin < 阈值 | reject |
| supplyChainFeasibility = 不可落地 | reject |
| complianceRisk = 极高 | reject |
| sizeWeightBand = 超大件 且 价格过低 | reject |

---

## 3. Step 1：RPS_base（基础分 0-100）

### 新公式

```text
RPS_base =
0.25 * review_growth_score +
0.10 * bsr_score +
0.20 * anti_monopoly_score +
0.15 * competition_ease_score +
0.20 * rating_window_score +
0.10 * review_base_score
```

### 为什么这样改
- **BSR 降权到 10%**：因为当前没有 `bigCategory`，BSR 不够稳
- **rating_window 提到 20%**：因为“有痛点且可切入”对选品价值更高

---

## 4. 子分规则

## 4.1 review_growth_score（30天评论增长，权重 25%）

| 30天评论增长 | 分数 |
|--------------|------|
| <= 0 | 10 |
| 1 - 10 | 25 |
| 11 - 30 | 55 |
| 31 - 50 | 75 |
| 51 - 100 | 100 |
| > 100 | 100 |

> 不再因为过热直接降分，过热改成标签。

---

## 4.2 bsr_score（BSR，权重 10%）

### v1 仅作辅助，不作重判

| BSR | 分数 |
|-----|------|
| <= 100 | 60 |
| 101 - 1000 | 90 |
| 1001 - 5000 | 100 |
| 5001 - 20000 | 80 |
| 20001 - 50000 | 45 |
| > 50000 | 20 |

> 注：在补上 `bigCategory` 前，BSR 不该承担太大权重。

---

## 4.3 anti_monopoly_score（反垄断分，权重 20%）

| 头部垄断程度 | 分数 |
|--------------|------|
| 低 | 100 |
| 中 | 60 |
| 高 | 20 |
| 极高 | 0 |

---

## 4.4 competition_ease_score（竞争容易度，权重 15%）

| 竞争激烈度 | 分数 |
|------------|------|
| 低 | 100 |
| 中 | 70 |
| 高 | 30 |
| 极高 | 10 |

---

## 4.5 rating_window_score（评分黄金区间，权重 20%）

| 当前评分 | 分数 | 含义 |
|----------|------|------|
| 3.8 - 4.2 | 100 | 黄金切入区 |
| 4.3 - 4.5 | 60 | 有机会但不明显 |
| < 3.7 | 40 | 可能有结构性硬伤 |
| > 4.6 | 20 | 市场成熟、改良空间小 |
| 空值 | 40 | 暂不重判，后续待补证 |

---

## 4.6 review_base_score（评论基数，权重 10%）

| 评论数 | 分数 |
|--------|------|
| < 100 | 100 |
| 100 - 500 | 85 |
| 501 - 2000 | 60 |
| 2001 - 5000 | 25 |
| > 5000 | 10 |

---

## 5. Step 2：固定风险扣分（替代乘数）

### 新逻辑
风险不再乘法放大，而是固定扣分。

| overallRisk | 扣分 | 说明 |
|-------------|------|------|
| 低 | 0 | 正常 |
| 中 | 8 | 有风险，但不该毁掉好品 |
| 高 | 15 | 保留排序参考，但应人工复核 |
| 极高 | reject | 直接淘汰 |

### 额外规则
如果 `overallRisk = 高`：
- 即使分数高，也不应直接进“供应链核价”
- 默认最多进入“观察池”或“待补证”

---

## 6. Step 3：人工 Override

人工允许干预，但不能打穿模型。

| 初筛结论 | 调整 |
|----------|------|
| 强烈看好 | +10 |
| 看好 | +5 |
| 先观察 | 0 |
| 清库存红海 | -15 |
| 明确不看好 | -10 |

### 留痕要求
- 修改理由
- 修改时间
- 本地备注

> 本地工具没有完整账户体系，v1 先不强制“谁改的”，改成建议在导出 session 时一起带出备注。

---

## 7. Step 4：不再扣分，改打标签

## 7.1 过热拥挤
条件：满足以下 4 条中的任意 2 条
- reviewGrowth30d > 100
- reviewCount > 2000
- headMonopoly in [高, 极高]
- competitionLevel in [高, 极高]

输出：
- Tag = `过热拥挤`

## 7.2 创新空间锁死
条件：
- currentRating > 4.6
- reviewCount > 2000
- 高频差评点为空或极弱

输出：
- Tag = `创新空间锁死`

## 7.3 数据过期
条件：
- `today - researchDate > 15天`

输出：
- Tag = `数据过期`
- UI 置灰
- 默认不进高优列表

## 7.4 待补证
条件：
- 核心字段缺失较多

输出：
- 不直接扣分
- 状态优先进入 `待补证`

---

## 8. 最终公式

```text
RPS_final = clamp(
  RPS_base + manual_override - fixed_risk_penalty,
  0,
  100
)
```

---

## 9. 数据完整度不再折损总分，改成状态分流

### 核心字段
- currentPrice
- currentBsr
- currentRating
- reviewCount
- reviewGrowth30d
- overallRisk

### 规则
| 缺失个数 | 处理 |
|----------|------|
| 0-1 | 正常排序 |
| 2-3 | 进入待补证 |
| >= 4 | 低优先级显示 + 待补证 |

### 说明
信息不足，不该偷偷折分，应该明确告诉用户：

> **这不是差品，是证据不够。**

---

## 10. 动作分层

| 条件 | 结果 |
|------|------|
| Gate reject | 淘汰库 |
| RPS_final >= 78 且非高风险 且非待补证 | 供应链核价 |
| 60 - 77 | 观察池 |
| 40 - 59 | 待补证 |
| < 40 | 淘汰库 |

### 特殊覆盖
- `overallRisk = 高` → 最高只能到 `观察池`
- `数据过期` → 默认不进高优推荐
- `待补证` → 优先进入 `待补证` 而不是观察池

---

## 11. v1 最重要的假阳性标签

| 标签 | 含义 |
|------|------|
| 高需低利陷阱 | v2 需要利润字段 |
| 创新空间锁死 | 市场成熟且无痛点 |
| 过热拥挤 | 有热度但很挤 |
| 高危禁入 | 不该继续投入 |
| 数据过期 | 结论可能已经失效 |
| 待补证 | 证据不足，别急着拍板 |

---

## 12. 这版规则的核心哲学

不是找“最完美的产品”，而是：

> **卖得动、能切入、风险没爆、而且现在就值得排在前面看。**
