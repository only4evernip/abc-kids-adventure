# WORKFLOW.md - Master

## 目标

定义 Master 的日常运行流程，确保它不是一堆静态文档，而是一套可以每天重复执行的三层联动分析流程。

核心要求：
- 固定顺序
- 固定输入
- 固定输出
- 固定降级规则

Master 每次运行都必须遵守：

1. **先看 Level 1：环境**
2. **再看 Level 2：主线**
3. **最后看 Level 3：个股**
4. **上层结论必须约束下层**

---

# 一、日常运行模式

建议 Master 采用两种模式：

## 模式 A：日线主流程（MVP 必做）

适合当前 v1：
- **收盘后更新**
- **盘前输出决策建议**

### 目标
- 用日线数据完成环境判断
- 更新主线 / 次主线 / 退潮方向
- 生成次日关注清单

## 模式 B：盘中增强流程（后续做）

后续扩展：
- 10:00 快照
- 11:30 快照
- 14:00 快照
- 收盘后总结

### 目标
- 动态观察龙头炸板
- 动态观察主线强弱变化
- 动态识别修复 / 退潮 / 回流

当前建议：
**先把模式 A 做稳，再考虑模式 B。**

# 二、日线主流程（推荐标准流程）

## Step 1：抓取市场环境数据（Level 1 原料）

### 输入数据
- 上证 / 深成指 / 创业板 / 沪深300 / 中证1000 等指数
- 两市成交额
- 上涨 / 下跌家数
- 涨停 / 跌停 / 炸板数据
- 连板高度
- 北向资金（如有）

### 处理动作
- 计算环境分数
- 判断环境标签：进攻 / 试错 / 混沌 / 防守
- 判断情绪周期：冰点 / 修复 / 主升 / 高潮 / 强分歧 / 退潮
- 标记是否存在系统性风险

### 输出结果
产出 `market_snapshot`

---

## Step 2：抓取板块与题材数据（Level 2 原料）

### 输入数据
- 板块涨跌幅
- 板块成交额
- 板块涨停数
- 板块龙头股
- 板块中军股
- 板块后排跟风情况

### 处理动作
- 判断当前唯一主线 / 双主线 / 无主线
- 判断板块阶段：启动 / 发酵 / 主升 / 强分歧 / 高位震荡 / 退潮 / 退潮反抽
- 判断主线健康度
- 标记退潮方向和危险方向

### 输出结果
产出 `theme_snapshot[]`

---

## Step 3：抓取候选个股数据（Level 3 原料）

### 输入数据
- 候选池股票 OHLCV
- 均线结构
- 涨停 / 炸板 / 封单强度（若有）
- 个股所属题材
- 是否与当前主线匹配

### 处理动作
- 判断个股角色：龙头 / 中军 / 补涨 / 跟风 / 杂毛 / 反抽股
- 判断是否顺势、顺主线、顺周期
- 识别当前信号：分歧低吸 / 回踩确认 / 打板确认 / 观察 / 退出信号
- 生成信号分数与动作建议

### 输出结果
产出 `stock_snapshot[]`

---

## Step 4：执行跨层约束

这是 Master 最关键的一步。

### 规则
- 如果 Level 1 = 防守，则 Level 2 与 Level 3 统一降级
- 如果 Level 2 = 退潮 / 退潮反抽，则 Level 3 禁止美化后排个股
- 如果个股与环境、主线冲突明显，则强制下调结论

### 处理结果
输出最终综合动作：
- 回避
- 观察
- 小仓试错
- 重点跟踪
- 可择机建仓
- 减仓
- 退出

---

## Step 5：生成标准输出

### 输出 1：市场总览
必须包含：
- 环境标签
- 情绪周期
- 当前主线
- 次主线
- 退潮方向
- 今日适合打法
- 今日严禁打法

### 输出 2：板块分析
必须包含：
- 板块名称
- 当前阶段
- 龙头 / 中军 / 后排结构
- 健康度判断
- 建议动作

### 输出 3：个股清单
建议至少输出：
- Top 3 重点跟踪
- Top 3 小仓试错
- Top 3 高风险回避
- 候选池全量排序（可选）

# 三、推荐的每日节奏

## 收盘后（主分析时段）
建议执行：
1. 更新指数与情绪数据
2. 更新主线与退潮方向
3. 更新候选股角色与信号
4. 生成完整分析日报

## 次日盘前（行动指引时段）
建议执行：
1. 回看前一日结论
2. 强调今日适合打法
3. 给出重点观察方向
4. 给出高危回避方向

## 盘中（后续增强）
可选执行：
1. 观察龙头是否炸板
2. 观察中军是否失速
3. 观察退潮是否扩散
4. 观察是否出现新主线穿越

---

# 四、输入与输出规范

## 输入顺序（必须固定）

### 输入层 1：市场快照
- 指数
- 成交额
- 涨跌停
- 炸板率
- 上下家数

### 输入层 2：板块快照
- 主线板块
- 板块阶段
- 龙头 / 中军 / 跟风结构

### 输入层 3：个股快照
- 个股角色
- 均线结构
- 当前信号
- 风险提示

## 输出顺序（必须固定）

1. 先给市场环境结论
2. 再给板块主线结论
3. 最后给个股动作结论

禁止反过来先说个股。

# 五、候选池生成建议

Master 的候选池不要太大。

## 推荐候选池来源
- 当前主线板块前排股
- 当前主线中军股
- 新启动方向的核心首板 / 二板
- 趋势保持完整的机构票

## 不建议纳入候选池的对象
- 与主线无关的杂毛
- 退潮板块中的反抽票
- 明显高位爆量后的后排补涨
- 没有成交额支撑的蹭概念小票

## 候选池规模建议
- 每日重点池：10-20 只
- 深度分析池：3-8 只
- 回避名单：3-10 只

# 六、日常运行中的强制检查项

每次输出前，Master 必须自检：

## 自检问题 1
**当前环境真的允许我推荐出手吗？**

## 自检问题 2
**这个板块真的是主线，还是只是脉冲热点？**

## 自检问题 3
**这只股真的是核心，还是跟风 / 杂毛 / 反抽？**

如果任何一个问题答不清楚，必须降级结论。

---

# 七、异常处理规则

## 异常 1：指数与个股赚钱效应背离
- 指数红盘，但中证1000 和高位股大跌
- 处理：标记为失真行情，优先保护资金

## 异常 2：板块看起来很强，但中军不动
- 处理：警惕一日游，降低主线评级

## 异常 3：个股很强，但板块已退潮
- 处理：优先按反抽处理，不轻易追高

## 异常 4：多题材轮动太快，主线不清
- 处理：环境降级为混沌，减少推荐数量

# 九、核心工作流一句话总结

Master 每天都应该按这条线运行：

**先看市场能不能做 → 再看哪里最值得做 → 最后才看具体做谁。**

如果顺序反了，Master 就会退化成普通股评机器人。
