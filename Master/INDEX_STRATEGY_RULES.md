# INDEX_STRATEGY_RULES.md - 指数策略规则骨架

## 目标

把"指数策略"从 Master 的短线情绪/主线判断里拆出来，单独做成一套 **宽基 ETF / 防守配置 / 调仓建议** 规则系统。

一句话：

> **环境定总仓，风格定核心，纪律定调仓。**

这套策略不追求抓最强个股，核心目标是：
- 控制回撤
- 稳定暴露市场 beta
- 在强环境里适度进攻
- 在弱环境里优先活下来

---

## 一、策略边界

### 它负责什么
- 宽基 ETF / 指数基金配置
- 真防守 / 权益防守 / 核心仓 / 卫星仓分桶
- A 股 / H 股 权重分配
- 调仓建议与再平衡节奏

### 它不负责什么
- 个股买卖点
- 打板 / 连板 / 题材接力
- 盘中高频判断
- 单一热点题材追涨

### 当前默认输出
1. 今天环境档位
2. 今天总权益仓位建议
3. 今天四大配置桶权重
4. 今天 A 股 / H 股 分配
5. 今天是否需要调仓
6. 若调仓，调哪些桶

---

## 二、核心配置桶

### 1. 真防守池
用于承接绝对防守需求。

默认工具：
- 货币基金
- 短债基金
- 政策性金融债指数基金
- 同业存单指数基金

原则：
- 真防守 = 流动性 + 低波动
- 不把高股息误当现金替代

### 2. 权益防守池
用于保留权益暴露，但优先控制波动。

默认工具：
- 红利低波 ETF
- 红利价值 ETF
- 高股息 ETF
- 央企价值 ETF

原则：
- 防守环境里，优先用权益防守替代激进宽基
- 不把"高分红"简单等于"低风险"

### 3. 核心仓
用于承接长期主配置。

默认工具：
- 沪深300 ETF
- 中证 A500 ETF
- 中证500 ETF（按环境启用）
- 恒生指数 ETF
- 恒生国企 ETF

原则：
- 核心仓追求可持续暴露，不追热点弹性
- 趋势未确认时，H 股核心保持克制

### 4. 卫星仓
用于提高弹性，但必须严格受环境约束。

默认工具：
- 恒生科技 ETF
- 中概互联网 ETF
- 中证1000增强 / 中小盘增强
- 高景气行业主动基金（仅在强环境）

原则：
- 卫星仓默认低配
- 只有环境强 + 主线清晰 + 波动可控时才扩张

## 三、环境分档

> **2026-03-11 重构**：从投机情绪指标改为 Beta 动量 + 广度 + 量能相对强度

### 核心指标（新版）

| 指标 | 说明 | 数据来源 |
|------|------|----------|
| turnover_ratio | 成交额 / 20日均线成交额 | Akshare / 东方财富 |
| up_ratio | 上涨家数 / 总家数 | Akshare |
| ma60_above_ratio | MA60以上个股占比 | Akshare / Wind |

### 环境 1：进攻
**触发条件（需同时满足）**：
- 成交额相对强度 >= 1.3x（量能放大 30% 以上）
- 上涨家数占比 >= 65%
- MA60以上个股占比 >= 60%

默认建议：
- 总权益仓位：70% - 85%
- 真防守：15% - 30%
- 权益防守：15% - 25%
- 核心仓：30% - 40%
- 卫星仓：5% - 15%

### 环境 2：试错
**触发条件（需同时满足）**：
- 成交额相对强度 >= 1.0x
- 上涨家数占比 >= 50%
- MA60以上个股占比 >= 45%

默认建议：
- 总权益仓位：45% - 60%
- 真防守：40% - 55%
- 权益防守：20% - 30%
- 核心仓：20% - 30%
- 卫星仓：0% - 8%

### 环境 3：混沌
**触发条件**：
- 不满足进攻/试错条件
- 但也不触发防守条件

默认建议：
- 总权益仓位：25% - 40%
- 真防守：60% - 75%
- 权益防守：15% - 25%
- 核心仓：10% - 20%
- 卫星仓：0% - 5%

### 环境 4：防守
**触发条件（满足任一）**：
- 跌停家数 >= 20
- 上涨家数占比 < 35%
- MA60以上个股占比 < 30%

默认建议：
- 总权益仓位：0% - 20%
- 真防守：80% - 100%
- 权益防守：0% - 15%
- 核心仓：0% - 10%
- 卫星仓：0%

### 旧版指标（已废弃，仅作风险提示）

以下指标不再作为环境分档的主判断依据，但会出现在风险提示中：
- ~~连板高度~~
- ~~炸板率~~
- ~~涨停家数绝对值~~

**废弃原因**：这些是投机情绪指标，与宽基 Beta 关联度低，容易导致"情绪狂热但指数滞涨"的错配。

## 四、A 股 / H 股 分配原则

### 默认原则
- A 股是主战场
- H 股作为风格补充，不是默认重仓方向

### 分配规则
#### 进攻环境
- A 股 / H 股 = 65 / 35 左右
- 若 H 股趋势未确认，回落到 75 / 25

#### 试错环境
- A 股 / H 股 = 70 / 30 左右
- H 股卫星默认收缩

#### 混沌 / 防守环境
- A 股 / H 股 的争论次要
- 先整体降权益，再谈区域分配

## 五、权重映射表

### 1. 标准权重映射（默认版）

| 环境 | 总权益 | 真防守 | 权益防守 | 核心仓 | 卫星仓 | A股/H股 |
|------|--------|--------|----------|--------|--------|---------|
| 进攻 | 80% | 20% | 20% | 45% | 15% | 65 / 35 |
| 试错 | 55% | 45% | 25% | 25% | 5% | 70 / 30 |
| 混沌 | 35% | 65% | 20% | 15% | 0% | 75 / 25 |
| 防守 | 10% | 90% | 10% | 0% | 0% | 100 / 0 |

### 2. 使用说明

- **总权益** = 权益防守 + 核心仓 + 卫星仓
- **真防守** 与总权益合计应为 100%
- **A股 / H股** 只作用于权益部分，不作用于真防守池
- 若环境强但 H 股趋势未确认，优先压缩 H 股，不优先压缩 A 股核心

### 3. 默认映射解释

#### 进攻
- 允许较高权益暴露
- 但仍保留 20% 真防守，避免把组合彻底推满
- 卫星仓开放到 15%，但仍不能喧宾夺主

#### 试错
- 这是当前最常用的过渡档
- 核心仓维持中等暴露
- 卫星仓只象征性开放，主要作用是保留弹性，不是重仓进攻

#### 混沌
- 默认关闭卫星仓
- 保留少量核心仓观察市场方向
- 权益防守优先于核心扩张

#### 防守
- 默认不做核心仓与卫星仓进攻
- 若保留权益，只通过权益防守池留少量暴露
- H 股在该阶段默认降为 0

### 4. 调仓触发规则

#### 环境跨档
- 进攻 ↔ 试错 / 试错 ↔ 混沌 / 混沌 ↔ 防守
- 视为 **一级调仓**
- 允许整体再平衡

#### 同档内微调
- 不改总框架
- 只在以下范围内微调：
  - 真防守：±5%
  - 权益防守：±5%
  - 核心仓：±5%
  - 卫星仓：±3%
  - A股/H股：±5%

### 5. 一票否决条件

出现以下任一情况时，卫星仓直接归零：
- 市场进入防守档
- 高位负反馈明显扩散
- 主线连续两天大分歧
- 宽基跌破关键均线且无修复信号

出现以下任一情况时，H 股权重默认降到 20% 以下：
- 港股趋势弱于 A 股
- H 股核心指数持续弱于 A 股核心宽基
- 市场进入混沌 / 防守档

## 六、调仓规则

### 1. 状态确认期（2026-03-11 Step 2 新增）

**核心思路**：只有连续 N 天（默认 2 天）信号一致，才确认跨档。

**实现逻辑**：
- 当天信号和当前确认状态一致 → 持续天数 +1
- 当天信号和当前状态不一致 → 进入"待确认"状态，仍维持原档位
- 连续 2 天收到相同新信号 → 确认跨档

**示例**：
```
Day 1: 当前"试错"，收到"进攻"信号 → 待确认，仍维持"试错"
Day 2: 再次收到"进攻"信号 → 确认跨档到"进攻"
Day 3: 收到"混沌"信号 → 待确认，仍维持"进攻"
Day 4: 再次收到"混沌"信号 → 确认跨档到"混沌"
```

**输出字段**：
```json
{
  "state_machine": {
    "confirmed_label": "试错",
    "days_in_state": 5,
    "pending_label": null,
    "state_changed": false
  }
}
```

### 2. EWMA 仓位平滑（2026-03-11 Step 2 新增）

**核心思路**：实际仓位 = 昨日仓位 × 70% + 目标仓位 × 30%，单日变动不超过 10%。

**参数**：
- α = 0.3（目标权重占比）
- 单日最大变动 = 10%

**示例**：
```
目标：真防守从 45% → 20%（目标降 25%）
实际：真防守从 45% → 37.5%（实际降 7.5%）
次日：37.5% → 32.25%（继续降）
...
约 3-5 天完成大切换
```

**输出字段**：
```json
{
  "smoothing": {
    "applied": true,
    "alpha": 0.3,
    "max_daily_change": 10,
    "daily_changes": {
      "true_defensive_weight": {
        "prev": 45,
        "target": 20,
        "actual": 37.5,
        "delta": -7.5
      }
    }
  }
}
```

### 3. 先看是否跨档
- 环境跨档：允许大调仓（但受 EWMA 平滑约束）
- 同档内部：只做微调

### 4. 调仓顺序
调仓顺序：
1. 先收/放卫星仓
2. 再调整核心仓
3. 再调整权益防守
4. 最后才动真防守底仓

### 5. 真防守不是频繁交易仓
- 真防守应作为底盘，不应高频进出
- 只有环境明显改善或恶化才显著调整

## 七、Step 3 新增机制（2026-03-11）

### 1. 组合偏离度触发再平衡

**核心思路**：
- 不再用简单的"单桶变化 5%/10%"判断
- 改用组合整体偏离度：`Tracking Error = sum(abs(当前权重 - 目标权重))`
- 只有偏离度 > 12% 才触发再平衡

**示例**：
```
目标：真防守 45%，核心仓 25%，卫星仓 5%
实际：真防守 50%，核心仓 22%，卫星仓 3%
偏离度 = |50-45| + |22-25| + |3-5| = 5 + 3 + 2 = 10%
结论：10% < 12%，不触发再平衡，避免小幅度频繁调仓
```

**输出字段**：
```json
{
  "tracking_error": {
    "value": 10.0,
    "threshold": 12,
    "should_rebalance": false,
    "bucket_deviations": {...}
  }
}
```

### 2. 最大回撤熔断机制

**核心思路**：
- 如果组合净值从高点回撤超过 8%，强制进入防守档 14 天
- 熔断期间不响应环境信号，只做防守
- 守住绝对收益底线

**触发条件**（简化版）：
- 连续 3 天防守档信号
- 或组合净值回撤 > 8%（需要外部数据支持）

**输出字段**：
```json
{
  "circuit_breaker": {
    "active": true,
    "days_remaining": 10,
    "reasoning": "连续 3 天防守档，触发熔断"
  }
}
```

### 3. 动态 ETF 优选（Smart Beta）

**核心思路**：
- 不固定死具体的 ETF，而是固定"桶的特性"
- 根据近期表现（夏普比率、最大回撤）动态选择池内最优 ETF
- 谁当前表现好，谁上位

**简化版实现**：
- 根据当前环境返回推荐的 ETF 列表
- 真实场景应接入 ETF 行情数据评分

**输出字段**：
```json
{
  "etf_selections": {
    "equity_defensive": {
      "recommended": [
        {"code": "红利低波", "weight": 0.4, "reason": "长期稳健"},
        {"code": "黄金", "weight": 0.3, "reason": "对冲股债双杀"},
        {"code": "央企价值", "weight": 0.3, "reason": "估值安全"}
      ],
      "reasoning": "基于当前环境，推荐权益防守配置..."
    }
  }
}
```

## 九、三步改进总结

| Step | 改进内容 | 核心效果 |
|------|----------|----------|
| **Step 1** | ETF 池扩充 + 阈值相对值 | 解决信号与资产错配，黄金对冲股债双杀 |
| **Step 2** | 状态确认期 + EWMA 平滑 | 防止单日信号闪烁，断崖式调仓变渐进式 |
| **Step 3** | 偏离度触发 + 熔断 + ETF 优选 | 避免小幅度频繁调仓，守住绝对收益底线 |

**改进前的问题**：
- 用投机情绪指标（连板、炸板）判断宽基配置
- 硬编码绝对阈值，缩量周期永远锁死
- 仓位从 80% 直接跳到 10%，断崖式调仓
- 单日信号闪烁导致频繁交易
- 没有熔断机制，指标失灵时深套

**改进后的效果**：
- 用 Beta 动量 + 广度 + 量能相对强度判断
- 阈值自适应市场水位
- 大切换分 3-5 天完成，单日最多变动 10%
- 连续 2 天才跨档，避免信号闪烁
- 回撤超 8% 强制防守 14 天，守住底线

## 十、当前决策

当前建议：

> **指数策略与 Master 保持"强关联、弱耦合"。**

也就是：
- 可以复用 Master 的环境判断
- 但输出层和调仓规则应独立维护
- 不要把短线题材判断和指数配置混成一个系统
