# ROADMAP.md - Master

## 目标

定义 Master 的版本路线，防止项目越做越散、越做越重。

Master 的原则不是一口气做成全自动量化终端，
而是按顺序把最值钱的能力先做稳：

1. 先把三层判断做准
2. 再把评分和样例跑稳
3. 再接数据与日报
4. 最后再考虑盘中增强与仓位系统

---

# 一、当前状态

目前已完成：
- `AGENT.md`：Agent 定位、三层逻辑、非目标
- `PROMPT.md`：模型行为规则
- `SCHEMA.json`：结构化输出框架
- `SCORING.md`：三层评分与降级规则
- `SAMPLES.md`：典型样例校准
- `DATA.md`：数据层设计
- `WORKFLOW.md`：日常运行流程

这意味着：
**Master 已经完成了 v1 文档级架构设计。**

但还没有完成：
- 实际数据抓取
- 自动标签生成
- 日报输出产物
- 真实案例回放验证

# 二、版本路线总览

## v1：文档化规则系统（已基本完成）

### 目标
把 Master 从想法压成一套明确的规则系统。

### 交付
- 三层联动框架
- 提示词
- 评分规则
- 样例校准
- 数据层设计
- 工作流

### 结果
可以作为：
- 继续喂 Gemini / 其他模型的母本
- 后续做脚本、日报、半自动系统的规格说明书

---

## v1.1：日报级可运行版本（下一阶段优先）

### 目标
让 Master 先以“日线级日报系统”跑起来。

### 需要补的东西
- 日报模板
- 候选池模板
- 黑名单 / 回避名单模板
- 基础数据抓取方案
- 主线 / 次主线 / 退潮方向输出逻辑

### 最小可运行结果
每天至少能产出：
1. 市场环境判断
2. 主线 / 次主线 / 退潮方向
3. Top 候选池
4. 高风险回避名单

### 不追求
- 盘中秒级分析
- 自动下单
- 超复杂回测

---

## v1.2：真实案例校验版

### 目标
用历史案例和近期实盘案例验证 Master 有没有明显跑偏。

### 需要补的东西
- `samples/` 目录拆分
- 进攻期案例
- 混沌期案例
- 退潮期案例
- 龙头 / 中军 / 跟风 / 杂毛对照案例
- 错判案例复盘

### 核心价值
不是为了写复盘文章，而是为了验证：
- Master 会不会把反抽当主升
- 会不会把杂毛当核心
- 会不会在混沌期过度激进
- 会不会在进攻期过度保守

---

## v2：半自动运行版

### 目标
把 Master 从“规则文档”推进到“半自动生产系统”。

### 需要补的东西
- 数据抓取脚本
- 结构化字段生成
- 自动评分器
- 主线识别流程
- 日报自动生成

### 输出形式
可以考虑生成：
- Markdown 日报
- JSON 结构化结果
- 候选池列表
- 风险警报列表

### 关键原则
v2 仍然以“日线稳定”为先，不着急上盘中。

---

## v2.1：盘中增强版

### 目标
增加盘中快照能力，提高对情绪突变和龙头炸板的响应速度。

### 建议增加的能力
- 10:00 / 11:30 / 14:00 快照
- 龙头炸板监控
- 中军失速监控
- 退潮扩散监控
- 新主线穿越监控

### 风险提醒
这一步很容易把系统复杂度拉爆。
必须在 v2 日线系统稳定后再做。

---

## v3：仓位管理与风险预算系统

### 目标
把 Master 从“告诉你做什么”升级到“告诉你做多少”。

### 需要补的东西
- 仓位上限规则
- 环境与仓位映射
- 主线强度与仓位映射
- 个股级别与仓位映射
- 组合风险控制
- 连续错误后的强制降仓机制

### 目标输出
例如：
- 防守期：0-10% 仓位
- 混沌期：10-25% 仓位
- 试错期：20-40% 仓位
- 进攻期：50-100% 仓位

---

## v4：高级增强（后期）

### 可选方向
- 监管 / 异动公告风险模块
- 分钟级大单净流入模型
- 拥挤度与筹码风险模型
- NLP 板块热度模型
- 更复杂的穿越主线识别

### 原则
这些都不是 Master 的起点能力。
如果基础判断都没做准，越加越乱。

# 四、MVP 定义

Master 的 MVP 不是全自动系统。

## MVP 只要求做到 4 件事
1. 判断当前市场环境（进攻 / 试错 / 混沌 / 防守）
2. 判断当前主线 / 次主线 / 退潮方向
3. 判断候选个股的角色与风险
4. 输出可执行的动作建议

## MVP 不做的事
- 自动下单
- 盘中超短决策
- 高复杂量化回测
- 追求覆盖所有板块和所有票

# 五、开发顺序建议

推荐按这个顺序推进：

1. `AGENT.md`
2. `PROMPT.md`
3. `SCHEMA.json`
4. `SCORING.md`
5. `SAMPLES.md`
6. `DATA.md`
7. `WORKFLOW.md`
8. `ROADMAP.md`
9. 日报模板
10. 候选池模板
11. 数据抓取方案
12. 真实案例目录

这个顺序的核心逻辑是：
**先定义脑子，再定义嘴巴，再定义数据，最后才定义手脚。**

# 六、成功标准

Master 后续是否算成功，不看文档写得多漂亮，而看这几件事：

1. 能否稳定识别进攻 / 混沌 / 退潮环境
2. 能否稳定区分真主线与一日游
3. 能否稳定区分龙头 / 中军 / 跟风 / 杂毛
4. 能否在退潮期明显减少错误出手
5. 能否在主升期不轻易错过核心机会

如果这 5 件事做不到，后面的盘中增强、仓位管理都没有意义。

# 七、一句话路线图总结

Master 的正确进化路径应该是：

**先做一个会过滤错误的市场总控台，再做一个会发现机会的交易辅助系统，最后才做一个会管理仓位的高级决策引擎。**
