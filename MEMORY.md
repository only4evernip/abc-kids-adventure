## 💡 核心认知框架（傅盛）

### 大语言模型 ≠ 大脑

| 组件 | 类比 |
|------|------|
| 大语言模型 | 智商水平 |
| 记忆系统 | 海马体 |
| Skill | 肌肉记忆 |
| Cron | 生物钟 |
| 多通道接入 | 感知器官 |

### 核心原则

1. **"你认为你记住的东西，那是你的幻觉。不靠脑子，靠文件。"**
2. **凡重要的事，必须写进机制里、写进文件里**
3. **用员工的思路，不是用工具的思路**

### 老板思维

- 知道目标
- 拆解任务
- 分配给最合适的人（或 AI）
- 检查结果、迭代调整

**差距不在于你有多聪明，而在于你懂不懂调度。**

### 安全红线（傅盛实战）

1. **保密守则**：发送前问自己"说出去会让老板尴尬吗？"
2. **权限控制**：十次有八次说"需要老板授权"
3. **先观察再授权**：新功能先验证，再开放实际操作权限

### 记忆架构（字节跳动 OpenViking）

**分层加载 L0/L1/L2**：
- L0 索引：目录，token 极少
- L1 摘要：按需读取
- L2 全文：只在需要时加载

**生命周期 P0/P1/P2**：
- P0 永久：核心规则
- P1 90天：项目信息
- P2 30天：临时信息

**关键**：在 AGENTS.md 写清楚什么时候查记忆、查哪个文件

## 🤖 Agent 模型评估框架（2026-03-09）

### 三大核心能力
评估模型是否适合做 Agent 基座，看三个维度：

| 维度 | 重要性 | 说明 |
|------|--------|------|
| 代码能力 | ⭐⭐⭐ | 能否准确执行任务，是 Agent 的"粗壮大腿" |
| 世界知识 | ⭐⭐⭐ | 能否理解业务场景，说人话而不是天书 |
| 多模态理解 | ⭐⭐ | 能否与现实交互，视觉+代码齐飞 |

### 成本控制的重要性
技术再好，用不起也是白搭：
- **Claude Opus 4.6**：封杀第三方工具，只能用 API，太贵
- **OpenAI**：支持第三方调用订阅额度，开放策略赢得开发者社区

### 工具搜索 vs 全量加载
OpenAI 的工具搜索思路（GPT-5.4）：
- 不一次性加载所有工具定义
- 轻量列表 + 按需搜索
- 减少 47% token 使用
- 类似 Skills 的渐进式披露

**结论**：这是上下文工程的最佳实践。

### GPT-5.4 = OpenClaw 天选模型
- 代码能力 = GPT-5.3-Codex
- 世界知识 > GPT-5.2
- 可用 $20/月 Codex 订阅额度
- 100万 token 上下文（超 27万 算 2 倍）

### GPT-5.4 接入 OpenClaw 配置指南

**两种接入方式**：

| 方式 | 认证类型 | 计费 | 适用场景 |
|------|----------|------|----------|
| Codex OAuth | ChatGPT 账户登录 | 走订阅额度 | 有 Plus/Pro 订阅的用户 |
| OpenAI API Key | 标准 API 密钥 | 按量计费 | 需精确控制成本的用户 |

**方式一：Codex OAuth（推荐）**
```bash
# 1. 安装 Codex CLI
npm install -g @openai/codex

# 2. 运行向导授权
openclaw onboard --auth-choice openai-codex

# 3. 切换默认模型
openclaw config set agents.defaults.model.primary "openai-codex/gpt-5.4"

# 4. 重启网关
openclaw gateway restart
```

**方式二：OpenAI API Key**
```bash
openclaw onboard --auth-choice openai-api-key
# 或
openclaw onboard --openai-api-key "你的API密钥"
```

配置文件设置：
```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "openai/gpt-5.4"
      }
    }
  }
}
```

**重要注意事项**：
1. **Codex 节点需要 OAuth Token**，不是普通 API Key
2. **auth-profiles.json** 必须包含 `"version": 1`、`"type": "token"`、`"expires"` 字段
3. **账户权限**：
   - 免费用户：有 Codex 权限（活动结束后可能降低）
   - Plus：可用 GPT-5.4 Thinking
   - Pro ($200/月)：可用 GPT-5.4 Pro
4. **上下文计费**：超过 272K token 的部分输入价格翻倍

## 🤖 e-commerce-scout 算法升级（2026-03-09）

### 核心算法

**动态权重调整机制**：
- 引入 MMC（市场成熟度系数）
- 平台/类目/市场阶段三维调节
- Softmax 归一化

**智能风险识别系统**：
- 五维动态风险评分卡
- 熔断机制（得分 < 60）
- 新增：流量成本刺客、生命周期陷阱、供应链波动

**竞争格局分析**：
- HHI 指数（市场集中度）
- CR4 指数（Top 4 集中度）
- 蓝海指数（BOI）公式
- 资本玩家识别（三大特征）

### 关键认知

1. **动态 > 静态**：静态权重无法适应不同市场阶段
2. **隐藏风险致命**：高阶玩家死于流量成本/趋势见顶/供应链波动
3. **垄断识别重要**：HHI > 2500 直接放弃
4. **量化决策**：BOI 指数一键判断蓝海/红海
